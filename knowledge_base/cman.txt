【命令】shutdown
【说明】是一个安全关闭或重启Linux系统的命令，在系统关闭之前会通知所有登录用户，系统即将关闭，此时新用户都不可以登录。
【常见用法】
        1. 关机：shutdown -h now （推荐）
	2. 重启：shutdown -r now （推荐）

【注意事项】
	关机：init 0
	重启：init 6

【其他】
	1. 1min 后关机：shutdown -h +1
	2. 11:00 重启：shutdown -r 11:00

【命令】reboot、halt、poweroff
【说明】关机与重启命令。从RedHat或CentOS 6开始，halt、poweroff、reboot 这三个命令对应的都是同一个man帮助文档，而halt和poweroff命令是reboot命令的链接文件。
【常见用法】
        1. 关机：halt、poweroff （立即关闭系统，并且需要关闭电源）
	2. 重启：reboot （推荐）
	3. 关机：init 0
	4. 重启：init 6

【注意事项】
	1. reboot --halt	等价 halt
	2. reboot --power-off	等价 poweroff

【其他】
	1. 查看链接文件：ll /sbin/halt
	2. 查看链接文件：ll /sbin/poweroff

【命令】pwd
【说明】显示当前目录的绝对路径。
【常见用法】
        1. pwd、pwd -L （忽略软链接的指向目标）
	2. pwd -P （显示软链接的指向目标）

【注意事项】
	1. pwd -L 等价 echo $PWD

	[root@ufo ~]# cd ./B; ll
	lrwxrwxrwx 1 root root        1 Dec 27 15:34 B -> A
	
	[root@ufo B]# pwd
	/root/B

	[root@ufo B]# pwd -L
	/root/B

	[root@ufo B]# echo $PWD
	/root/B

	[root@ufo B]# pwd -P
	/root/A

【其他】
	1. 软链接 <=> 软链接文件 <=> 符号连接，标识类型都是l，存储的是文件指向目标就是文件，存储的是目录指向的目标就是目录）。
	2. 如何区分软链接文件存储的是文件还是目录？不能直接通过标识符l来区分。

	[root@ufo ~]# file A
	A: directory

	[root@ufo ~]# file B
	B: symbolic link to A

【命令】cd
【说明】获取帮助信息，作为man、help 命令的补充，很少用到。
【常见用法】
        1. 家目录：cd （默认）、cd ~、echo $HOME
	2. cd -P （如果目录是一个软连接，直接切换到软链接指向的真正物理目标目录）
	3. cd -L （如果目录是一个软连接，直接切换到软连接所在目录）
	4. cd - 、echo $OLDPWD （用户上一次所在的目录）
	5. cd .. 、cd ../../ （上一级目录）

【注意事项】
	1. tab 键自动补齐（如果是目录自动补齐/）
	2. 相对路径不是从"/"开始、绝对路径是从"/"开始
	3. 家目录：/root/、/home/用户名/

【命令】tree
【说明】以树形结构显示目录下的内容。
【常见用法】
        1. tree -a （包含所有隐藏文件，以 "." 点开头的文件）
	2. tree -d （只显示目录）
	3. tree -f （显示每个文件的全路径）
	4. tree -i （不显示树枝，常与 -f 参数配合使用）
	5. tree -L level （遍历目录的最大层数，level 为大于0的正整数）
	6. tree -F （结尾加上标识符号，执行文件*、目录/、Socket文件=、符号链接@、管道名称| 、等等）
	7. tree --noreport （不显示最后一行统计信息）

【注意事项】
	1. yum -y install tree
	2. export LANG=en_US.UTF-8  （调整字符集，防止树形结构显示乱码; 与linux字符集和连接linux客户端的字符集都有关联）

【命令】touch
【说明】创建空文件或改变文件的时间戳属性 
【常见用法】
	1. touch -a （更改指定文件的最后访问时间）
	2. touch A B C
	3. touch A{01..05} （序列：01-05）
	4. touch -m （更改指定文件的最后修改时间）
	5. touch -d 20231228 A （指定时间更新指定文件的修改时间）
	6. touch -r A B （B复制A文件的修改时间属性）
	7. touch -t 202312282230.50 （指定时间更新指定文件的修改时间）

【注意事项】
	1. touch 是创建空文件，mkdir 是创建空目录
	2. touch 不能创建目录，但是可以修改目录的时间戳
	3. stat 命令可以查看文件的时间戳属性（访问时间、修改时间、状态时间）
	4. ls -lt <=> ls --mtime 修改文件内容（modify time）
	5. ls -lc <=> ls --ctime 修改文件内容、移动文件、改变文件属性（change time）
	6. ls -lu <=> ls --atime 查看文件内容（access time）

【命令】ls
【说明】显示目录下的内容以及相关属性信息
【常见用法】
	1. ls -l <=> ll （使用长格式列出文件以及目录信息）
	2. ls -a （显示所有文件，包括以"."开始的隐藏字符）
	3. ls -t （根据文件修改时间[mtime]排序，默认以文件名排序）
	4. ls -r （以相反的顺序排序）
	5. ls -F （加上文件标识符*、/、=、@、|、等）
	6. ls -p （只在目录后边加/ ）
	7. ls -i （显示文件 inode 节点信息）
	8. ls -d B （如果遇到目录，则显示目录自身的信息，并不显示目录里边的信息，也不跟随符号链接）
	9. ls -h （以可读方式显示文件或目录大小，如1KB、234MB、2GB 等）
	10. ls -A （列出所有文件，但不包括"."与”.."这两个目录）

 

【注意事项】
	1. ls -lrt 、ls -rt （查看最近更新过的文件或目录）
	2. ls -lhi --time-style=long-iso (以可阅读的方式显示文件或目录详情)

【其他】
	--不常用选项：
	1. ls -S （根据文件大小排序）
        2. ls -R （递归列出所有子目录，相比tree会更好用一些）
        3. ls -x （以横向顺序显示目录中的文件和子目录。内容较多时，一行一行输出）
        4. ls -X （根据字母顺序排列。相同字母时，文件在前目录在后）
	5. ls -c （根据ctime时间排序）
	6. ls -u （根据atime时间排序）
	7. ls --color={never,always,auto} （不同文件类型显示不同颜色，never-不显示，always-显示，auto-自动）
	8. ls --full-time （以完整时间格式输出）
	9. ls --time-style={full-iso,long-iso,ios,locale} （以不同时间格式列出，long-iso效果最好，默认locale格式）
	10. ls --time={atime,ctime} （按照不同时间属性显示，默认以mtime时间显示）

【命令】cp
【说明】复制文件或目录
【常见用法】
	1. cp -p （复制文件时保持源文件的所有者、权限信息以及时间属性）
	2. cp -d （如果复制的文件是符号链接，仅复制符号链接本身，不复制符号链接指向的目标文件或目录，符号链接本身存储的内容是地址指向）
	3. cp -r （递归复制目录，复制所有层级的子目录及文件，不复制链接文件指向目标）
	4. cp -R （会递归复制链接文件的指向）
	5. cp -a <=> cp -pdr
	6. cp -i （覆盖已有文件前提示用户确认）
	7. cp -t （可颠倒顺序，默认"cp 源文件 目标文件"，使用-t 参数格式变为"cp -t 目标文件 源文件"）

【注意事项】
	1. alias cp='cp -i' （通常设置了别名） 
	2. which cp （/bin/cp 屏蔽别名）
	3. \cp （屏蔽别名）
	4. unalias cp （取消别名）
	5. ~/.bashrc （注释取消别名，重启也生效）（不推荐）
	6. cp /A/a.txt{,.bak} （快捷写法，等价 cp /A/a.txt /A/a.txt.bak ）

【命令】mv
【说明】移动或重命名文件
【常见用法】
	1. mv -f （不询问，强制覆盖已有目标）
	2. mv -i （询问目标是否覆盖）
	3. mv -t （颠倒顺序，"mv 源文件 目标文件" 变为 "mv -t 目标文件 源文件" ）
	4. mv -u （源文件比目标文件新，或目标文件不存在时才进行移动）


【注意事项】
	1. mv 可以移动文件和目录
	2. alias mv （一般存在别名）
	3. 如何屏蔽别名，可以参考 cp 命令
	4. 移动的同时可以"重命名"
	5. 为了规范或避免误操作，移动目录时注意是否需要加"/", Tab 键自动补全（非常重要！）

【其他】
	1. 相同路径下的文件可以重命名（文件）
	2. 将文件或目录移动到指定的目标目录之下（目录）
	   mv A B、mv A/ B、mv A/ B、mv A/ B/

【命令】rm
【说明】删除文件或目录
【常见用法】
	1. rm -f （强制删除。忽略不存在的文件，不提示确认）
	2. rm -i （删除前提示确认）
	3. rm -I （在删除超过三个文件或递归删除前要求确认）
	4. rm -r （递归删除目录及其内容）

【注意事项】
	1. alias rm='rm -i'

【其他】
	非常重要！！！
	1. 常用mv替代rm，不着急删除，先移动到回收站/tmp
	2. 删除之前务必做好备份，最好是异地备份
	3. find ./ -type f -name "*.txt" -mtime +7 | xargs rm -f
	4. find ./ -type f -mtime +7 -exec rm {}\;
	5. 尽量不使用通配符
	6. 尽量不使用 rm -rf 
	7. 注意：rm -f ./* 、rm -f * 、rm -f ./ * （注意"*"之前是否存在空格）
	8. 注意：rm -f A、rm -f A/ （如果A指向一个目录，一个是删除软连接文件、另一个是删除软链接指向的目标）
	9. 尽量不要使用 "rsync --delete"，无法用恢复工具ext3grep来恢复。
	10. 未雨绸缪总比已经删除了再解决要好的多。

【命令】ln
【说明】硬链接（hard link）与软链接（符号链接，symbolic link）
【常见用法】
	1. ln A B （无参数，创建硬链接。A源文件、B目标文件）
	2. ln -s A B （创建软链接，即符号链接。A源文件、B目标文件/目录）

【注意事项】
	1. 硬链接生成普通文件，标识符（-）
	2. 软连接生成符号链接文件，标识符（l）
	3. 无论创建软连接或符号链接，目标文件不能事先存在

	4. 硬链接：允许多个文件指向同一个索引节点（inode）;允许一个文件拥有多个有效路径名（多个入口）。可以建立硬链接到重要文件，以防止误删除源数据，只要最后一个硬链接不删除，就可以重做硬链接恢复文件（ln A B，类似复制文件）
	5. rm A 、rm A/ （删除硬链接或软链接、删除软链接的指向目标）（切记Tab自动补全"/"的含义，防止误删除）
	6. 软链接文件的inode值和源文件不同。软链接文件存放的是源文件的路径，指向源文件实体。删除软链接文件，源文件还在。删除源文件，软连文件接还在，但是已经无法访问了，相当于实际已经删除了数据，无法恢复。
	7. 创建硬链接或软连接可以使用绝对路径、相对路径。
	8. 创建硬链接（多个文件入口，不可以创建目录，不能跨文件系统），创建软连接（存储地址，可以指向文件或目录）
	9. 软链接可以跨文件系统，所以尽量限制跨文件，不易后期维护。

【其他】

	1. （不推荐）如何变相的扩容磁盘？A磁盘满了/A/data/，B磁盘没有满 /mnt/bdisk/data/，B磁盘比A磁盘空间大。
	-- A磁盘适当删除或移动一些数据，已便可以创建软连接
	-- A磁盘需要迁移的数据移动到B磁盘目录下（静态数据或关闭应用时），例如 cp -pdr /A/data/，/mnt/bdisk/data/
	-- 创建软连接文件/A/data，/A/data的内容指向/mnt/bdisk/data/A/data/ （ln -s /mnt/bdisk/data/A/data/ /A/data）
	-- 只是改变了数据存放的位置和访问方式，在实际应用中，这种方法通常用来重新组织文件结构或者方便程序读取共享数据，而不是解决磁盘容量问题。

【命令】find
【说明】查找目录下的文件。
【常见用法】
	1. 语法说明：find [-H,-L,-P] [path] expression[options,tests,actions]
	[-H,-L,-P]： 如何处理符号链接
	[path]：需要查找的路径
	[options,tests,actions]：参数、限定条件、执行动作
	
	2. options模块
	-depth：从指定目录下最深层的子目录开始查找
	-maxdepth levels：查找最大的目录级数，levels为自然数
	-regextype type：改变正则表达式的模式。默认为emacs，还有posix-awk、posix-basic、posix-egrep、posix-entended

	3. tests模块
	-mtime [-n,n,+n]：按照文件的修改时间来查找文件。-n更改时间距离现在n天以内，+n更改时间距离现在n天以前，n更改时间距离现在第n天。
	-ctime [-n,n,+n]：按照文件的改变时间来查找文件。单位为天。
	-amin：按照文件的访问时间来查找文件。单位为分钟。
	-cmin：按照文件的改变时间来查找文件。单位为分钟。
	-mmin：按照文件的修改时间来查找文件。单位为分钟。

	-user：按照文件属主来查找文件
	-group：按照文件所属的组来查找文件。

	-nouser：查找没有有效属主的文件，即该文件的属主在 /etc/passwd 中不存在
	-nogroup：查找没有有效用户组的文件，即该文件所属的组在 /etc/groups 中不存在

	-name：按照文件名查找文件，只支持*，？，[]等特殊通配符
	-newer：查找修改时间比指定文件新的文件
	
	-regex：按正则表达式
        -ireges：按正则表达式，不区分大小写

	-path pattern：指定路径样式，配合 -prune 参数排除指定目录
	-perm：按照文件权限来查找文件
	-size n[cwbkMG]：查找文件大小为[+n,n,-n]的文件（大于，等于，小于），单位[cwbkMG]
	-type：查找某一类型的文件。（块设备文件b,字符设备文件c,目录d,管道文件p,符号链接文件l,普通文件f,socket文件s,door是D ）

	4. actions模块
	-delete：将查找出的文件删除
	-exec：对匹配的文件执行该参数所给出的shell命令
	-ok：和exec作用相同，但是会在执行每个命令之前让用户确定是否执行
	-prune：跳过匹配到的目录，即不对其进行进一步的查找（排除一些目录，通常配合path使用）
	-print：将匹配的文件输出到标准输出（默认功能）
	
	5. OPERATORS：find支持逻辑运算符
	!：取反，不满足条件的文件或目录
	-a：取交集，全拼为and
	-o：取并集，全拼为or

【注意事项】
	1. find . ! -name "*.txt"
	2. find . -name "*.txt" -a -size +1M
	3. find . -name "*.txt" -o -name "*.doc"

	4. find . -regex ".*\.txt"
	5. find . -iregex ".*\.txt"

	6. find mydir -name "subdir1" -prune -o -print
	7. find . -path "./exclude" -prune -o -name "*.txt" -print

	8. find . -newer reference.txt （查找比指定文件新的） 
	9. find . -perm /u=rw （查找有读写权限的）

	10. find . -name "link.txt" （默认，link.txt是目录下的一个文件。如果find过程中链接文件存储的是目录，直接输出实际目录）
	11. find -H . -name "link.txt" （默认同上）
	12. find -L . -name "link.txt" （会跟随符号链接的指向，即输出实际目录，又输出链接目录）
	13. find -P . -name "link.txt" （不跟随符号链接的指向，只输出实际目录，忽略链接链接文件存储是目录的）

	14. find . -atime -2
	15. find . -type d
	16. find . -perm 755 （权限是755的）
	17. find . \( -path ./A -o -path ./B \) -prune -o -print （忽略多个目录）
	18. find . -maxdepth 1 -type d ! -name "." -o -name "BBB" （"."以外的目录或"BBB"文件）
	
	19. find正则语法：find pathname -regextype "type" -regex "pattern" （-name 支支持"*","?","[]"三个通配符，遇到复杂的匹配需求，就需要使用正则）
	find . -regex ".*/find"
	find . -regextype "posix-egrep" -name "test[0-9]"

	20. find . -type f -exec ls -l {} \; （不同系统";"含义不同，加"\"进行转义）
	find . type f -mtime +14 -exec rm {} \;
	find . type f -mtime +14 -ok -exec rm {} \; （y删除，n不删除）

	find . -type f | xargs ls -l （xargs可以接收很多参数，并向其他命令传递参数）
	find . -name "*.txt" | xargs -i mv {} ./A/ （-i参数使得{}可替代find找到的内容）
	find . -name "*.txt" | xargs -p rm -f （y删除，n不删除）
	find . -type f -exec echo {} \; （每次获得一个结果就输出一次）
	find . -type f | xargs -i echo {} （输出结果只有一行，xargs获取到所有文件名一次性输出）
	find . -type f | xargs -n 3 -i echo {}	（每次指定3个参数）

	find+exec （文件多时效率低）
	find+xargs （推荐）
	命令+`find...` （优先执行倒引号，方便理解）

【其他】
	1. find+exec
	exec将查找的结果文件名逐个传递给后边的命令执行，如果文件比较多，执行效率会比较低；文件名有空格或特殊字符时，无法正常失败
	2. find+xargs
	xargs将查询的结果文件名一次性传递给后边的命令执行，命令执行效率高，可以使用-n 参数控制一次传递文件的个数；处理特殊文件名，可以采用特殊方式（find . -name "*ufo*" -print0 | xargs -0 ls -lh）

【命令】cat
【说明】合并文件或查看文件内容
【常见用法】
	
	五大常用场景：
	1. cat file.txt 查看文件内容 
	2. cat file1.txt file2.txt > newfile.txt 把多个文件合并成一个
	3. cat > file 、cat >> file 覆盖或追加文件内容，输入结束，Ctrl+D 可结束编辑。（了解）

	5. cat /dev/null >file.txt （清空文件内容）

	1. cat -n 显示行号
	2. cat -b 显示行号，但是会忽略空白行号

	7. cat -t <=> cat -vT
	8. cat -T 将Tab字符显示为^I
	9. cat -v 将非打印字符显示为可见字符的转义序列（更清晰的展示特殊字符或控制字符）

	2. 上一条也可以当做脚本内容，通过sh 执行脚本追加操作。

	3. cat <<END    （可以当做脚本中的菜单模块，整体可以放到一个菜单函数中）
	   菜单选项...
	   END

	4. cat 是针对多行文本内容进行编辑或在已有内容结尾追加新内容。echo 单行显示或追加比较方便
	5. cat | mysql -uroot -pufo <<EOF
	   sql语句...
	   EOF

	1. -d	指定合并的分隔符号，默认是Tab
	2. -s	每个文件占用一行

	3. paste AAA BBB
	4. paste -d: AAA BBB
	5. paste AAA （显示文件内容）
	6. paste -s AAA （将多行内容视为多个字段，转换成1行）
	7. paste -s AAA BBB （每个文件内容占用1行）

【注意事项】
	1. 输入如果是"-"，则表示从标准输入读取内容。
	2. cat AAA BBB 将文件按顺序前后合并
	3. paste -sd '=\n' （先用"="分隔，再用"\n"分隔。每两个字段合并为一行）
	4. paste -d '=' - - < file.txt （每个"-"代表读入一行，行之间用"="分隔）
	5. xargs -n 2 < file.txt |sed 's/# #=#g' （每两个参数视为一行，默认空格分隔）
	6. sed 'N;s#\n#=#g' file.txt （只有多行模式的"\n"才可以被替换）

【命令】sort
【说明】文本排序。将输入的文件内容按照指定的规则进行排序，然后将排序结果输出。
【常见用法】
	1. -b	忽略每行开头存在的空格字符
	2. -n	依照数值的大小进行排序（一般纯数字，混合字母也不会报错，也可以混合排序后再截取数字部分）
	3. -r	倒序排序（默认从小到大，反转后从大到小）
	4. -u	去重复（默认会排序后，然后去掉重复，包括空行重复）
	5. -t	指定分隔符
	6. -k	指定区间排序

【注意事项】
	1. 默认以行为单位进行排序（第1个字段），默认从行首字符向后，依次按ASCII码值进行比较，输出默认按照升序进行排列。
	2. sort file.txt （默认转先换成ASCII码值，再对比后排序）
	3. sort -t "" -k2 （指定空格分隔，以第2个字段排序）
	4. sort -n -t "" -k3,3 -k4.1,4.3 file.txt （先按第3个字段排序，再按第4个字段的第1个字符到第三个字符排序）
	5. "." 连接字符，"," 连接字段，可以锁定区间进行排序（字段，字段局部等）
	6. ASCII码值区分大小写，所以排序不考虑大小写问题。

	6. --max-depth=N		显示N级别子目录的大小，当N=0时，等价-s 选项

【注意事项】
	1. du -sh ./*	（推荐）
	2. du -sh --max-depth=1 ./* （只显示第一层目录的大小）
	3. du -sh --max-depth=1 --exclude-from="./A" ./* （排除指定目录，并仅展示第一层）

	场景一：
	du 统计整个目录大小100G，实际ls 查看没有大文件，lsof | grep del 确认有哪些正在删除文件的进程，kill 调对应进程。du 与 ls 查看大小一致了，一般删除大文件时，偶尔会出现。

	场景二：
	持续向一个文件写入数据，ls 查看文件大小持续变化，但是 du 查看文件大小不变化或只会在某个瞬间突然变化。这种现象可能与缓冲落盘有关。

	场景三：
	ls 查看文件系统分配的逻辑大小，du 查看文件实际占用磁盘大小。针对空洞文件两个命令查看到的大小可能不一致。

	场景四：
	空洞文件的形成：首先创建一个新文件，默认会先分配一个初始的逻辑大小，但是并不会立即在磁盘上分配相应的物理空间。如果写入数据的时候跳过一些位置，文件系统不会为这些跳过的空间分配实际的磁盘空间。相反，跳过的位置会形成一个空洞。 这样可能导致 ls 大于 du 看到的结果。空洞文件的作用：可以多线程从同一个文件的不同入口写入数据，提高了写入效率。

【命令】tar
【说明】打包备份。可以对多个文件进行打包，还可以对多个文件打包后进行压缩。打包是指将一大堆文件或目录变成一个总文件，压缩则是将一个大的文件通过一些压碎哦算法变成一个小文件。 
【常见用法】
	1. -z 	通过 gzip 压缩或解压
	2. -c 	创建新的 tar 包
	3. -v 	显示详细的 tar 命令执行过程
	4. -f 	指定压缩文件的名字
	5. -t 	不解压查看 tar 包的内容
	6. -p	保持原有文件的属性
	7. -P	以绝对路径打包，危险参数
	8. -j   通过 bzip2 命令压缩或解压
	9. -x   解开 tar 包
	10. -C  指定解压的目录路径
	11. --exclude=PATTERN	打包时排除不需要处理的文件或目录
	12. -X 文件名		从指定文件读取不需要处理的文件或目录
	13. -N 日期		仅打包比指定文件日期新的文件，可以用于增量打包备份
	14. -h			打包软连接文件指向的真实源文件
	15. --hard-dereference  打包硬链接文件
	
【注意事项】
	1. tar -czvf aaa.tar.gz ./A/    （打包、压缩）
	2. tar -xzvf aaa.tar.gz		（解包、解压）

	3. tar -tzvf aaa.tar.gz		（不解压，仅查看压缩包中的内容）
	4. tar -czvf aaa.tar.gz ./A/ --exclude=./A/file.txt （排除指定文件或目录，如果排除的是目录，结尾不要加"/"，否则不会成功。排除两个以上的目录可以并列使用多个 --exclude 。太多也可以使用-X选项。打包用相对路径，排除用绝对路径，则可能排除不会成功。）
	5. tar -czvfh aaa.tar.gz ./A/	（包括打包软链接指向的目标）
	6. tar -czvf aaa.tar.gz `find /etc/ -type f` （打包压缩所有普通文件）
	7. tar -xzvf aaa.tar.gz -C /root/B （解压到指定目录）

【其他】
	1. 打包用相对路径，排除用绝对路径，则可能排除不会成功。
	tar -czvf aaa.tar.gz ./A/ --exclude=/root/A/file.txt
	2. 打包用相对路径，排除用相对路径，成功。
	tar -czvf aaa.tar.gz ./A/ --exclude=./A/file.txt
	tar -czvf aaa.tar.gz ./A/ --exclude=file.txt
	3. 打包用绝对路径，排除用绝对路径，成功。
	tar -czvf aaa.tar.gz /root/A/ --exclude=/root/A/file.txt
	4. 打包用绝对路径，排除用相对路径，成功。
	tar -czvf aaa.tar.gz /root/A/ --exclude=./A/file.txt
	tar -czvf aaa.tar.gz /root/A/ --exclude=file.txt

	总结：相对~相对、绝对~任意

	2. 打包目录之前一般先进入到目录的上一级，然后再打包。少数情况下使用绝对路径打包，解压缩要考虑是否会覆盖原文件。

【命令】gzip
【说明】压缩或解压文件。gzip 用于将一个大文件通过压缩算法变成一个小文件。gzip 命令不能直接压缩目录，因此目录需要先用 tar 命令打包成一个文件，然后再用 gzip 进行压缩。
【常见用法】
	1. -d 		解开压缩文件
	2. -v 		显示指令执行过程
	3. -l		不解压，列出压缩文件的压缩信息

	6. -数字<1-9>	指定压缩率，默认为6，值越大压缩率越高

	4. gzip -c aaa1.txt > aaa.txt.gz        （不推荐）（压缩，源文件保留）
	   gzip -dc aaa.txt.gz > aaa2.txt	（不推荐）（解压，源文件保留）

	5. gzip 套件包含了许多可以直接处理压缩文件的实用程序。
	   zcat   ~  cat   ~  gzip -dc 
	   zgrep  ~  grep
	   zless  ~  less  ~  zmore
	   zdiff  ~  diff

	   gunzip ~  gzip -d

【命令】scp
【说明】远程复制文件。scp 命令用于再不同主机之间复制文件，采用SSH协议来保证复制的安全性。scp 每次都是全量完整复制，因此效率不高，适合第一次复制时使用，增量复制建议使用 rsync 命令。
【常见用法】

	2. 从远程服务器复制到本地服务器（文件或目录）。第一次 scp 和 SSH 第一次登录一样，要输入远程及其的密码。（拉取：host1 <- host2）

	scp -rp [[user2@]host2:]file2 ./

	1. 注意：文件比较大时，全量文件传输容易中断。

【命令】rsync
【说明】文件同步工具。rsync 是一款开源的、快速的、多功能的、可实现全量以及增量的本地或远程数据镜像同步备份的优秀工具。rsync 适用于 unix、linux、windows 等多种操作系统平台。
【常见用法】

	本地模式：
	rsync [option] [src] [dest]

	远程shell模式：
	拉取：rsync [option] [user@]host:src [dest]	
	推送：rsync [option] [src] [user@]host:dect

	rsync守护进程模式：（持续同步）
	拉取：rsync [option] [user@]host::src [dest]
	拉取：rsync [option] rsync://[user@]host[:port]/src dest

	推送：rsync [option] [src] [user@]host::dect
	推送：rsync [option] [src] rsync://[user@]host[:port]/dest


	-v,--verbose          详细模式输出，传输时的进度等信息
	-z, --compress        传输时进行压缩以提高传输效率，--compress-level=NUM可按级别压缩
	-a, --archive         以递归方式传输文件，并保持所有文件的属性，相当于-rtopgD1
	-r, --recursive       对子目录以递归模式，即目录下的所有目录都以同样的模式传输，注意是小写r
	-t, --times           保持文件的时间信息
	-o,--owner            保持文件的属主信息
	-p, --perms           保持文件的权限
	-g, --group           保持文件的属组信息
	-P --progress         显示同步的过程及传输时的进度等信息
	-D,--devices          保持设备文件信息
	-l,--links            保留软链接
	-e,--rsh=COMMAND      使用的信道协议，指定替代 rsh的 shell程序，例如:ssh
	-n                    测试选项，模拟执行
	--exclude=PATTERN     指定排除不需要传输的文件模式(可以模糊匹配)
	--exclude-from=FILE   从文本文件读取需要排除的文件列表
	--bwlimit=KBPS        限制传输速度
	--delete              使目标目录内容和源保持目录一致，删除不同的文件


【注意事项】
	1. 可以跨不同系统复制，但是首先要安装 rsync 工具，并且网络是连通的。
	
	2. 类似于 cp ， 但是比 cp 好的一点是可以实现增量复制。
	rsync -av /data1/ /data2	（如果目录结尾带"/"，就会复制目录下的内容，而不复制目录本身）
	rsync -av /data1  /data2	（不带"/"，目录本身和目录下的内容都会复制）

	3. 删除文件
	rsync -av --delete /null/ /tmp/ （null 是空目录，不同的文件或目录会被删除。null 中有什么内容，tmp 中就会有什么内容，与null对比后，删除tmp中多余的内容）

	4. 类似于 scp。使用rsync重复复制时，直至目录下文件相同就不再进行复制了。（推荐使用）
	rsync -av ip:/tmp/ /tmp			（明文拉取）
	rsync -av /tmp/ ip:/tmp/		（明文推送）

	5. 使用SSH隧道模式
	rsync -av -e "ssh -p 22" ip:/tmp/ /tmp	（加密拉取）
	rsync -av -e "ssh -p 22" /tmp ip:/tmp/	（加密推送）


	6. 注意：如果没有其他业务占用带宽，可以不使用 -z 选项。不压缩传输，几乎可以满带宽传输。如果压缩传输，则网络发送素服就会骤降，压缩的速率赶不上传输的速度。
	7. 可以使用 --bwlimit 选项限制带宽，避免过度占用带宽。
	8. -avn	可以模拟传输的过程，如果没有报错，则可以去掉n选项真正的传输文件。

【其他】
	1. 复制中断时，可以重复执行，直到文件一致，停止复制。

【命令】useradd
【说明】创建新用户或更改用户信息。
【常见用法】
	
	1. 添加用户的原理：
	在使用useradd命令时，若不加任何参数选项，后面直接跟所添加的用户名，那么系统首先会读取/etc/login.defs (用户定义文件)和/etc/default/useradd (用户默认配置文件)文件中所定义的参数和规则，然后根据所设置的规则添加用户，同时还会向/etc/passwd (用户文件)和/etc/group (组文件)文件内添加新用户和新用户组记录，向/etc/shadow (用户密码文件)和/etc/gshadow (组密码文件)文件里添加新用户和组对应的密码信息的相关记录。同时系统还会根据/etc/default/useradd文件所配置的信息建立用户的家目录，并将/etc/skel中的所有文件（包括隐藏的环境配置文件）都复制到新用户的家目录中。

	2. useradd 不加 -D 选项，创建用户：
	-c comment		新用户password文件中的说明栏（冒号分隔后的第五列)。
	-d home_dir		新用户每次登入时所使用的家目录。
	-e expire_date		用户终止日期。日期的指定格式为YYYY-MM-DD。
	-f inactive_days	用户过期几日后永久停权。当值为0时用户立即被停权，而当值为-1时则关闭此功能，预设值为-1。
	-g initial_group	指定用户对应的用户组。用户组名必须为系统现已存在的名称。
	-G group,[...]		定义此用户为多个不同组的成员。每个用户组使用逗号(,)分隔。用户组名同-g选项的限制。默认值为用户的起始用户组。
	-m			用户目录如不存在则自动建立。
	-M			不建立用户家目录，优先于/etc/login.defs文件设定。创建虚拟用户时一般不需要建立家目录，部署应用服务时则需要创建虚拟用户。
	-n			默认情况下用户的用户组与用户的名称是相同的。如果命令添加了-n参数，就不会生成与用户同名的用户组了。
	-r			此参数是用来建立系统用户的。系统用户的UID会比定义在系统档上/etc/login.defs.的 UID_MIN 要小。注意此用法下uscradd所建立的用户不会建立用户家目录，也不会在乎记录在l/etc/login.defs.中的定义值。如果需要用户家目录必须额外指定-m参数来建立系统用户。这是Red Hat额外增设的选项。
	-s shell		用户登入后使用的Shell名称。默认值为不填写，这样系统会帮助指定预设的登入Shell（根据letc/default/useradd预设的值）。
	-u uid			用户的ID值。这个值必须是唯一的，除非用-o选项。数字不可为负值。

	3. useradd 添加 -D 选项，改变用户的预设值：
	当执行useradd带-D参数时，可以更改新建用户的默认配置值( /etc/default/useradd )或者由命令行编辑文件更改预设值。可简单理解该参数( -D)就是用于修改/etc/default/useradd配置文件的内容的，若这个文件的内容被修改，则添加新用户不加参数时默认值就会从该/etc/default/useradd中读取。

	-b default_homc		定义用户家目录的基本目录，当用户家目录不存在时，此目录将作为家目录生效。
	-e default_expire_date	用户账号停止日期，格式为YYYY-MM-DD，同 useradd 的-e参数。
	-f default_inactive	用户过期几日后停权。同useradd 的-f参数。
	-g default_group	新用户起始用户组名或ID。用户组名必须为现已存在的名称。用户组ID也必须为现已存在的用户组。同useradd 的-g 参数。
	-s default_shell	用户登入后使用的Shell名称。修改后新加入的用户都将使用此Shell类型，同useradd -s参数。

【注意事项】
	1. useradd ufo			（默认创建同名的用户、用户组）
	2. groupadd -g ufoAAA		（创建用户组）
	   useradd  -g ufoAAA -u  ufo	（创建用户，并指定用户组）
	3. id ufo			（查看用户和用户组的信息）
	4. useradd -M -s /sbin/nologin	（-M 不创建家目录，-s 指定用户登陆后的shell，此处/sbin/nologin代表禁止登录）

	5. 添加ufo用户，并设置用户注释信息为"SysUser"，UID指定为806，归属为用户组root,aa成员，其shell类型为/bin/sh，设置家目录为/tmp/ufo，用户过期时间为2024/12/12，过期2天后停权。
	useradd -u 806 -s /bin/sh -c "SysUser" -G root,aa -e "2024/12/12" -f 2 -d /tmp/ufo ufo

	6. /etc/passwd 文件中的ufo用户信息说明：
	ufo	:x	:806		:1002	:SysUser	:/tmp/ufo	:/bin/sh
	账号	密码	UID		GID	注释		:家目录		:shell解释器


	7. useradd -D <=> vim /etc/default/useradd
	useradd -D -s /bin/sh		（修改登录后的默认shell） 
	useradd -D -e "2024/12/12"	（修改用户的有效期）
	chage -l ufo 			（查看用户有效期）

【命令】passwd
【说明】修改用户密码。可以修改用户密码以及密码过期时间等内容。普通用户和超级用户都可以运行passwd命令，但是普通用户只能修改自身的用户密码，超级用户root则可以设置或修改所有用户的密码。
【常见用法】
	1. -k		更新已过期的用户有效期
	2. -l		锁定用户，被锁定的用户不能登录。仅root用户有权限。
	3. --stdin	从标准输入读取密码字符串。（此时，只需要输入1次密码）
	4. -u		解除对用户的锁定。仅root用户有权限。
	5. -e		使用户密码立即过期，将在用户下次登录时强制要求用户修改密码。仅root用户有权限。

	7. -x		设置修改密码的最长天数。仅root用户有权限。
	8. -w		设置用户在密码过期前收到警告信息的天数。仅root用户有权限。
	9. -i		设置密码过期多少天后禁用账户。仅root用户有权限。

	3. passwd ufo
	4. passwd -S ufo
	5. echo "123456" | passwd --stdin ufo
	6. 要求用户7天内不能修改密码，60天以后必须修改密码，过期前10天通知用户，过期30天后禁止用户登录。
	passwd -n 7 -x 60 -w 10 -i 30 ufo
	chage -l ufo

	7.创建10个账户，并将用户名和密码保存到文件中。 
	echo aa{01..10} | tr " " "\n" | sed -r 's#(.*)#useradd \1;pass=$((RANDOM+10000000));echo "$pass" | passwd --stdin \1;echo -e "\1 `echo "$pass"`" >> /tmp/ufo.log#g'

【命令】sudo
【说明】sudo命令可以让普通用户在执行指定的命令或程序上，拥有超级用户的权限。普通用户不需知道root密码就可以得到相关授权，这个授权可以用visudo来配置和管理。
【常见用法】
	
	-l	列出当前用户可以执行的命令。只有在sudoers文件里的用户才能使用该选项。
	-h	列出使用方法，并退出。
	-H	将环境变量中的HOME(家目录）指定为要变更身份的使用者家目录（如果不加-u参数就是系统管理者root)。
	-V	显示版本信息，并退出。
	-v	sudo在第一次执行时，或者在N分钟内没有执行(N预设为五），则会询问密码，这个参数用于重新做一次确认。
	-u	以指定用户的身份执行命令。后面是除root 以外的用户，可以是用户名，也可以是uid。
	-k	清除时间戳上的时间，下次再使用sudo 时要再输入密码。
	-K	与-k类似，同时还要删除时间戳文件。
	-b	在后台执行指定的命令。
	-p	可以更改询问密码时的提示语。
	-e	不执行命令，而是修改文件，相当于命令sudo edit。
	
	1. 提权后执行命令的格式为"sudo 命令"
	2. 切换到root执行相关操作"sudo su -"，注意此时提示的密码是当前用户的密码，而不是root的密码。
	3. sudo -l 可以查看当前用户被授予的sudo权限集合。

	4. su 与 sudo 的区别与联系：长期提权、临时提权。
	

	su         --->需要密码---\			            /---> 不需要密码（已配置NOPASSWD选项）
	su -	   --->需要密码----> 超级用户 ---> vim /etc/sudoers ----> sudo 命令、sudo su - 
	su - root  --->需要密码---/   (root)	      (visudo)      \---> 需要当前用户密码



                                                        -------------未被授权-----------------
                                                        |                                    |
                                                        |                                    v
	sudo 命令  ---> 判断时间戳是否过期？ ---> 判断是否被授权？ ---> 有权限则执行 ---> 退出sudo 
                           /var/db/sudo	           /etc/sudoers
				|                       ^
	                        v                       |
                        过期则输入用户自己密码 ---->-----


	5. sudo -k 、sudo -K 清空sudo用户的时间戳，这样还会提示输入密码，但是如果配置授权对应用户加了NOPASSWD选项，那么执行sudo命令时，永久不会提示输入密码了。

【命令】fsck
【说明】检查并修复linux文件系统中的错误。类似的命令还有 e2fsck 命令。
【常见用法】
        
	1. cat /etc/fstab
	2. fsck filesys （filesys 可以是一个设备名/dev/sda1，一个挂载点/usr，一个文件系统的磁盘标签，也可以是UUID）

	-a	自动修复文件系统，不询问任何问题
	-s	按顺序检查多个文件系统
	-t	指定要检查的分区的文件系统类型
	-A	依照/etc/fstab配置文件的内容，检查文件内所列的全部文件系统
	-N	不执行指令，仅列出实际执行会进行的动作 

【注意事项】
        1. 使用fdisk 要特别注意文件系统必须是卸载状态，否则可能会出现故障！！！
	2. 平时没有必要使用这个命令检查磁盘，只有当系统开机显示磁盘错误时，才需要执行。
	3. 不要对正常的分区使用fsck，在不加参数的情况下，fsck会根据/etc/fstab 进行文件系统检查，等价于 fsck -As 参数的功能。
	4. 系统开机之所以会通过fsck自检，就是因为系统开机过程中会优先读取/etc/fstab文件，当文件最后一列设置为1或2时，这个磁盘在开机时就会调用fsck进行自检。（检查自动挂载等情况）
	5. 一般划分磁盘分区时，最后一列设置为0，即开机不对磁盘进行检查。如果真有问题，可以在系统启动后人为进行检查。
	6. 当linux 系统遭遇突然断电等非正常关机时，很容易导致文件系统数据损坏，造成系统不能重新启动。根据启动流程提示，进行fsck修复。
	7. 谨慎使用！！！

【命令】sync
【说明】sync命令将内存缓冲区的数据强制刷新到磁盘。
【常见用法】
        
	linux 内核为了达到最佳的磁盘操作效率，默认会先在内存中将需要写入到磁盘的数据缓存起来，然后等待合适的时机将它们真正写入到磁盘中，这在绝大多数情况下都是没有任何问题的，但是如果系统出现宕机、掉电等情况，就有可能会导致有些文件内容没能保存下来。当系统正常关机或重启时，会将缓冲区中的内容自动同步到磁盘中。我们也可以手动执行 sync ，将内存中的文件缓冲内容强制写到磁盘中。

	通常情况下，没必要执行sync 这个命令，linux 内核会尽快让内存中的数据自动同步到磁盘上去。

【注意事项】
        
	1. sync （执行命令，并不会输出结果）

	x	高亮背景显示列

	z	打开、关闭字体颜色
	b	打开、关闭字体加粗
	
	c	切换完整程序名
	u	指定用户相关的进程
	n	指定输出多少行
	k	终止一个指定的pid进程
	
	Shift+<\>	切换排序列
	Ctrl+C	<=> q	退出top命令

【命令】strace
【说明】strace 是 linux 环境下的一款程序调试工具。strace 会跟踪程序运行时的整个生命周期，输出每一个系统调用的名字、参数、返回值和执行所消耗的时间等，是高级运维和开发人员排查问题的杀手锏。
【常见用法】
        
	-c	统计每一个系统调用所执行的时间、次数和出错的次数等
	-d	输出strace关于标准错误的调试信息
	-f	跟踪目标进程，以及目标进程创建的所有子进程
	-ff	如果提供-o filename，则将所有进程的跟踪结果输出到相应的filename.pid 中,pid是各进程的进程号
	-i	输出系统调用的入口指针
	-q	禁止输出关于脱离的消息
	-r	输出每一个系统调用的相对时间
	-t	在输出中的每一行前加上时间信息。例如16:45:28
	-tt	在输出中的每一行前加上时间信息，精确到微秒。例如11:18:59.759546
	-ttt	输出中的每一行前加上时间信息，精确到微秒，而且时间表示为UNIX时间戳。例如1486111461.650434
	-T	显示每次系统调用所花费的时间
	-v	对于某些相关调用，把完整的环境变量、文件stat结构等打印出来
	-x	以十六进制形式输出非标准字符串
	-xx	所有字符串以十六进制形式输出

	-e expr	输出过滤器，通过表达式，可以过滤掉你不想要的输出
		expr是一个表达式，用于控制如何跟踪:[qualifier=][!]value1[,value2]...
		说明:
		1. qualifier只能是trace、abbrev、verbose、raw、signal、read、write 其中之一
		2. value是用来限定的符号或数字
		3. 默认的qualifier是trace
		4. 感叹号是否定符号
		
		例如:
		-e open等价于-e trace=open，表示只跟踪open调用。而-e trace!=open表示跟踪除了open 以外的其他调用
		
		常见选项:
		-e trace=[set]只跟踪指定的系统调用
		-e trace=file只跟踪与文件操作有关的系统调用
		-e trace=process只跟踪与进程控制有关的系统调用
		-e trace=network只跟踪与网络有关的系统调用
		-e trace=signal只跟踪与系统信号有关的系统调用
		-e trace=desc只跟踪与文件描述符有关的系统调用
		-e trace=ipc只跟踪与进程通信有关的系统调用
		-e abbrev=[set]设定strace输出的系统调用的结果集
		-e raw=[set]将指定的系统调用的参数以十六进制显示
		-e signal=[set]指定跟踪的系统信号
		-e read=[set]输出从指定文件中读出的数据
		-e write=[set]输出写入到指定文件中的数据
		
-o filename	将strace的输出写入文件filename
-p pid		指定要跟踪的进程pid，要同时跟踪多个pid，重复多次-p 选项即可※
-s strsize	指定输出的字符串的最大长度，默认为32。并没有将文件名视为字符串，默认全部输出
-u username	以 username的UID和GID执行所跟踪的命令

【注意事项】
        
	strace -tt -f /bin/mysql		  （跟踪目标进程以及目标进程创建的所有子进程）
	strace -tt -f -e trace=ufo.txt /bin/mysql （只跟踪与文件操作有关的系统调用） 	
	strace -tt -f -e trace=ufo.txt -p 1909	  （只跟踪某个pid的）
	strace -c /bin/mysql			  （为进程的所有的系统调用做一个统计分析）
	strace -c -o mysql.log /bin/mysql 	  （-o 选项将结果输出到文件中）
	strace -T /bin/mysql			  （对系统调用进行计时）
	
	1. strace命令很适合处理程序僵尸、命令执行报错等问题，如果从程序日志和系统日志中看不出问题出现的原因，则可以 strace 一下，也许会有答案，不过也需要使用者有足够的耐心去查看输出!

【命令】ifconfig
【说明】ifconfig 命令用于配置网卡IP地址等网络参数或显示当前网络的接口状态，其类似于Windows下的ipconfig命令，这两个命令很容易混淆。此外，ifconfig命令在配置网卡信息时必须以root用户的身份来执行。如果系统中没有ifconfig命令，那就需要安装一下，安装命令为yum -y install net-tools 。
【常见用法】
        
	-a	显示所有网络接口信息
	up	激活指定的网络接口
	down	关闭指定的网络接口
	hw	设置网络接口的物理地址（MAC地址）

	1. ifconfig
	2. ifconfig eth0
	3. ifconfig -a
	4. ifconfig eth1 up
	5. ifconfig eth1 down
	6. ifconfig eth1 192.168.110.110          （配置临时ip）（生产环境一般不这样修改）
	7. ifconfig eth1:0 192.168.110.110/24 up  （配置网卡别名eth1:0）
	   ifconfig eth1:1 192.168.111.111/24 up  （配置网卡别名eth1:1）
	
【注意事项】
        
	1. linux下的网络接口名类似eth0、eth1、lo 等，分别表示第1块网卡、第2块网卡和回环接口。ifconfig 如果不加选项，则显示系统中所有的网卡信息。
	2. 使用ifconfig命令配置网卡信息仅会临时生效，重启网络或服务器配置就会失效。

【命令】route
【说明】显示或管理linux系统的路由表。route 命令设置的路由主要是静态路由。
【常见用法】
        
	路由的概念
	1. 源主机（数据传输） ---> 网络节点A ---> 网络节点B ---> ... ---> 目标主机
	2. 每一个网络节点就是一个路由，根据约定（路由规则）传输数据到下一个网络节点。

	路由分类
	1. linux 上配置的路由都属于静态路由。静态路由的规则是系统管理员使用route命令加入的。也就是通过手动输入的方式来加入的路由规则。
	2. 动态路由就是无须手动输入的路由规则，其路由规则是本机与不同的机器之间通过路由程序相互交换路由规则而来的。

【注意事项】

	-n	直接使用IP地址，不进行DNS解析主机名
	-ee	显示更详细的路由信息
	add	添加路由信息
	del	删除路由信息
	target	指定目标网络或主机。可以使用IP地址或主机/网络名
	-net	到一个网络的路由，后面接网络号地址
	-host	到一个主机的路由。后面接主机地址
	
	gw GW		为发往目标网络/主机的任何分组指定网关
	dev If		指定由哪个网络设备出去，后面接网络设备名，如eth0等
	netmask NM	为添加的路由指定网络掩码        

	1. route
	2. route -n	（不解析DNS，可以加快显示速度）
	3. route del default <=> route add default
	4. route del default gw 10.0.0.2 <=> route add default gw 10.0.0.2
	5. route -net 102.168.1.0/24 gw 10.0.0.254
	   route -net 102.168.1.0 netmask 255.255.255.0
	6. route -net 102.168.1.0 netmask 255.255.255.0 dev eth1
	   route -net 102.168.1.0/24 dev eth1

	7. 以上配置再重启网络时都会失效，如果希望永久生效，可以尝试如下方法：
	   touch /etc/sysconfig/network-scripts/route-eth1
	   #添加，适用于生产环境。（推荐）
	   192.168.1.0/24 via 10.0.0.254

	   touch /etc/sysconfig/static-routes
	   #添加
	   any net 192.168.1.0/24 gw 10.0.0.254
	
	   vim /etc/rc.local
	   #添加，只在开机时加载，手工重启网络后会失效，但是重启系统后会生效。
	   route add -net 192.168.1.0/24 gw 10.0.0.254

	8. 如果时配置默认路由网关，则可以写在配置文件里。
	   /etc/sysconfig/network-scripts/ifcfg-eth0
	   #修改
	   GATEWAY=...

	9. 配置和删除主机路由（就是去往某个主机地址的路由）
	   route add -host 192.168.2.13 dev eth2
	   route add -host 202.81.11.91 dev lo

	   route del -host 192.168.2.13 dev eth2

【命令】arp

	-v			显示详细的arp缓存条目，包括缓存区条目的统计信息

【注意事项】
        
	1. arp
	2. arp -n
	3. arp -n 172.26.63.253
	4. arp -s 172.26.63.253 ee:ff:ff:ff:ff:ff （静态绑定IP地址和MAC地址）
	5. arp -d 172.26.63.253 （删除静态绑定的ip）

【命令】ip
【说明】ip 命令是iproute软件包中的一个强大的网络配置工具。用于显示或管理linux系统路由、网络设备、策略路由和隧道。
【常见用法】
        
	1.选项很多，略

	2.显示网络设备属性
	ip link show dev eth1
	ip -s link show dev eth1
	ip -s -s link show dev eth1
	
	3.关闭和激活网络设备
	ip link set eth1 up
	ip link set eth1 down
	
	4.修改网卡MAC地址
	ip link set eth1 address 0:0c:29:13:10:11

	5.查看网卡信息
	ip a
	ip a show eth1

	6.添加或删除IP地址（可以添加多个IP，称为辅助IP）
	ip a add 172.16.1.12/24 dev eth1
	ip link set up
	ip a show eth1
	ip a del 172.16.1.12/24 dev eth1
	（删除网卡的主IP地址，则会同时删除该网卡的所有IP地址）
	（删除网卡的辅助IP地址，不会影响该网卡的其他IP地址）

	7.创建IP别名
	ip a add 172.16.1.12/24 dev eth1 label eth1:1
	
	8.使用ifconfig命令创建的别名IP，ip命令能够查看到；相反通过ip命令创建的辅助IP，ifconfig命令则查询不到，除非使用ip命令的label功能创建的别名IP。
	ifconfig

	9.查看路由表
	ip route

	10.添加或删除路由表
	ip route add 172.16.1.12/24 via 10.0.0.253 dev eth0
	ip route del 172.16.1.12/24

	11.查看arp缓存，添加或删除静态的arp项
	ip neighbour
	ip neighbour add 192.168.1.100 lladdr 00:0c:29:c0:5a:ef dev eth0
	ip neighbour del 192.168.1.100 dev eth0


【注意事项】
        
	辅助IP、别名IP、vip、浮动IP、虚拟IP 的区别是什么？

	ip 将逐渐取代 route 和 ficonfig 命令

【命令】ping
【说明】用于测试主机之间网络的连通性。执行ping命令会使用ICMP传输协议，发出要求回应的信息，若远端主机的网络功能没有问题，就会回应该信息，因而可得知该主机运行正常。
【常见用法】

	-t<生存期>	设置发送的数据包生存期（TTL）的值
	-w 截止时间	超过截止时间，立即退出ping程序
	-W 超时时间	等待响应的超时时间
	
【注意事项】
        
	1.显示一个时间作为衡量网络延迟的参数，以判断源主机与目标主机之间的网络质量。TTL值称为生存期，它是ICMP报文在网络上的存活时间。TTL值反应了报文所能经过的最多的路由器数目，TTL值变为0，则路由器将不再转发此报文。
	ping www.baidu.com
	ping -c 3 -i 3 -s 1024 -t 255 www.baidu.com

【命令】wget
【说明】wget命令用于从网上下载资料。支持断点下载，支持FTP和HTTP下载，支持代理服务器。在带宽很窄的情况下或不稳定的网络中有很强的适应性。如果是由于网络下载失败，wget会不断尝试，直到整个文件下载完毕。如果是服务器打断了下载过程，它会再次连接到服务器上从停止的地方继续下载。这对那些限定了连接时间的服务器上下载大文件非常有用。
【常见用法】
        
	-o		将命令的执行结果写入文件中。
	-O		指定保存的文件名后下载文件
	--limit-rate	限速下载

	-c		断点续传
	--user-agent	指定客户端标志
	-q		关闭下载时的输出
	--tries=number	设置重试的次数
	--spider	模拟爬虫访问
	-T seconds	设置访问的超时时间 <=> --timeout=seconds

【注意事项】
        
	1. 下载并更新文件名
	wget -O /etc/aaa.txt http://www.baidu.com/repo/file.txt

	2. 限速下载
	wget --limit-rate=3k -O /etc/aaa.txt http://www.baidu.com/repo/file.txt 

	3. 断点续传
	wget -c --limit-rate=3k -O /etc/aaa.txt http://www.baidu.com/repo/file.txt
	wget -c http://www.baidu.com/repo/file.txt

	4. 后台下载
	wget -b http://www.baidu.com/repo/file.txt

	5. 伪装代理名称下载
	wget --user-agent="代理名" http://www.baidu.com/repo/file.txt

	6. 监控网站URL是否正常。3秒超时，重试1次
	wget -q -T 3 --tries=1 --spider www.baidu.com
	echo $?		（返回状态码0，表示正常）

	7. 可以通过 python 开启内置的简易的http服务器。cd 进入需要传输文件的目录下，执行：
	python3 -m http.server [端口号]
	python -m SimpleHTTPServer [端口号]
	在网络内的其他设备上执行：
	wget -b -c http://102.168.22.33:8000/aaa.txt

【命令】tcpdump
【说明】tcpdump命令是一个截获网络数据包的包分析工具。tcpdump可以将网络中传送的数据包的”头“完全截获下来以提供分析。他支持针对网络层、协议、主机、端口等过滤，并支持与、或、非逻辑语句协助过滤有效信息。tcpdump 命令工作时要先把网卡的工作模式切换到混杂模式，所以要以root的身份运行。
【常见用法】
        
	-A		以ASCII码的方式显示每一个数据包（不会显示数据包中链路层的头部信息）。在抓取包含网页数塢的数据包时，可方便查看数据
	-c<数据包数目>	接收到指定的数据包数目后退出命令
	-e		每行的打印输出中将包括数据包的数据链路层头部信息
	-i<网络接口>	指定要监听数据包的网络接口

	-nn		不将协议和端口数字等转成名字
	-q		以快速输出的方式运行，此选项仅显示数据包的协议概要信息，输出信息较短
	-s<数据包大小>	设置数包抓取长度，如果不设置则默认为68字节，设置为0则自动选择合适的长度来抓取数据包
	-t		在每行输出信息中不显示时间戳标记
	-tt		在每行输出信息中显示无格式的时间戳标记
	-ttt		显示当前行与前一行的延迟
	-tttt		在每行打印的时间戳之前添加日期
	-ttttt		显示当前行与第一行的延迟
	-v		显示命令执行的详细信息
	-vv		显示此-v选更加详细的信息
	-vvv		显示比-vv选项更加详细的输出	

【注意事项】
        
	1. 精简输出
	tcpdump -q

	2. 指定数据包数目，不需要 Ctrl+C 推出
	tcpdump -c 5

	3. 监听指定网卡收到的数据包
	tcpdump -i eth0

	4. 不解析，加快显示
	tcpdump -n host 10.0.0.1

	5. 只监听收到的数据包
	tcpdump -n dst host 10.0.0.1

	6. 监听指定端口的数据包
	tcpdump -nn port 22

	7. 监听指定协议的数据包（tcp、udp、icmp、arp、ip）
	tcpdump -n arp
	tcpdump -n icmp

	8. 逻辑运算
	tcpdump -n ip host 10.0.0.12 and ! 10.0.0.1
	tcpdump tcp dst port 80 or src 12.130.132.30 -i eth0

【命令】eval
【说明】evel将参数当做命令执行。
【常见用法】
        
	[root@ufo ~]# echo `hostname -I`
	172.26.18.65
	[root@ufo ~]# echo $(hostname -I)
	172.26.18.65
	[root@ufo ~]# eval echo '`hostname -I`'		（优先解析执行单引号中的命令）
	172.26.18.65

【命令】history
【说明】history命令用于查看所执行的历史命令。
【常见用法】

	1. 默认显示数百条记录。重启关机后，内存中的命令不会记录。已经保存的不会丢失。
	history
	history 10	近10条
	history -d 990	删除历史命令（查看命令序号：history ）
	history -c	清楚所有历史命令记录

【命令】read
【说明】read 命令会从标准输入中读取一行，并把输入行的每个字段的值都指定给shell变量。
【常见用法】

	1. read 读取的值默认存储在REPLY变量中。
	echo $REPLY

	2. 读取变量
	read one
	echo $one

	read one two
	echo $one $two

	3. 提示语，输入值存储到变量age中。
	[root@ufo ~]# read -p "请输入年龄: " age
	请输入年龄: 200
	[root@ufo ~]# echo $age
	200

	4. 关闭回显，例如密码输入不显示。
	[root@ufo ~]# read -s -p "请输入年龄: " age
        请输入年龄: ...

	5. 设置输入字符的最大长度。
	read -n 3 -p "只能输入3个字符:" num

【注意事项】

【命令】type
【说明】type命令用于判断指定命令的类型。
【常见用法】
	
	-a	显示所有相关信息
	-t	file:普通命令;  alias:命令别名;  builtin:内置命令
	
【注意事项】

	[root@ufo ~]# type ls
	ls is aliased to `ls --color=auto'
	[root@ufo ~]# 
	[root@ufo ~]# type -a ls
	ls is aliased to `ls --color=auto'
	ls is /usr/bin/ls

	[root@ufo ~]# type cd
	cd is a shell builtin
	[root@ufo ~]# type -a cd
	cd is a shell builtin
	cd is /usr/bin/cd
	[root@ufo ~]# type -t cd
	builtin

【命令】ulimit
【说明】查看系统资源使用情况，同时也可以修改进程或用户等对系统资源分配的额度。
【常见用法】

	vim /etc/security/limits.conf	（永久生效需要修改配置文件）
